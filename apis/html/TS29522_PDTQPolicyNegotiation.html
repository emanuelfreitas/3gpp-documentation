Authorizations:
NoneoAuth2ClientCredentials
path Parameters
afId
required
string

Represents the identifier of the AF.

Request Body schema: application/json
required
One of
Array of objects (AltQosParamSet) non-empty

Contains the alternative QoS requirements as a list of individual QoS parameters sets in a prioritized order.

altQosRefs
Array of strings non-empty

Contains the alternative QoS requirements as a list of QoS References in a prioritized order.

appId
string (ApplicationId)

String providing an application identifier.

aspId
required
string

Contains the identifier of the application service provider.

required
Array of objects (TimeWindow) non-empty

Contains the time interval(s).

object (LocationArea5G)

Represents a user location area when the UE is attached to 5G.

notifUri
string (Uri)

string providing an URI formatted according to IETF RFC 3986.

numberOfUEs
required
integer

Contains the number of UEs.

Array of objects (PdtqPolicy) non-empty

Contains the PDTQ policies.

object (QosParameterSet)

Contains the QoS requirements expressed as one or more individual QoS parameters.

qosReference
required
string

Contains the requested QoS requirements expressed as a QoS Reference.

referenceId
string (PdtqReferenceId)

Represents a PDTQ Reference ID.

selectedPolicy
integer

Contains the identifier of the selected PDTQ policy. This attribute shall not be present during resource creation, it may be provided only in a subsequent resource update.

warnNotifEnabled
boolean

Indicates whether the PDTQ warning notification is enabled (true) or not (false). Default value is false when this attribute is omitted.

suppFeat
string (SupportedFeatures) ^[A-Fa-f0-9]*$

A string used to indicate the features supported by an API that is used as defined in clause 6.6 in 3GPP TS 29.500. The string shall contain a bitmask indicating supported features in hexadecimal representation Each character in the string shall take a value of "0" to "9", "a" to "f" or "A" to "F" and shall represent the support of 4 features as described in table 5.2.2-3. The most significant character representing the highest-numbered features shall appear first in the string, and the character representing features 1 to 4 shall appear last in the string. The list of features and their numbering (starting with 1) are defined separately for each API. If the string contains a lower number of characters than there are defined features for an API, all features that would be represented by characters that are not present in the string are not supported.
